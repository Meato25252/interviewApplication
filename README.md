# 簡単な面接練習アプリ(Android)
## 概要

本アプリは、想定質問に対して音声で回答することで、面接練習を行うAndroidアプリです。
質問の読み上げから回答の音声入力、結果の保存までを一連の流れとして実装しています。

本アプリでは「動作すること」だけでなく、挙動の検証を重ねながら仕組みを理解することを重視して開発しました。

## 主な機能

* 質問の表示および音声読み上げ（TextToSpeech）
* 音声による回答入力（SpeechRecognizer）
* 制限時間付きの回答（CountDownTimer）
* 回答内容の保存・参照（Room）
* 質問一覧の表示（RecyclerView）

## 使用技術

* Java
* Android SDK
* 音声認識（SpeechRecognizer）
* 音声合成（TextToSpeech）
* ローカルデータベース（Room）
* RecyclerView
* CountDownTimer

## 工夫した点
### 1. ライフサイクル

音声認識や音声合成は状態を持つため、
アクティビティの状態に応じて適切に制御する必要があります。

そのため、

* onPauseで音声認識の停止
* onDestroyでリソースの解放

を行い、不正な状態を防ぐようにしました。

### 2. 自力での実装

本アプリはAIに頼らず、
公式ドキュメントや動作検証をもとに実装しました。

特に音声認識周りは挙動が複雑であるため、
エラーや状態遷移を確認しながら開発を進めました。

## 課題
### 1. 責務の分離が不十分

現在はActivityに処理が集中しており、
音声処理・タイマー・データ処理が密結合になっています。

今後は以下を検討しています。

* 音声処理のクラス分離
* データ層の整理

### 2. 音声認識の制約

音声認識の仕様により、以下の課題があります。

* はっきり発話しないと認識されにくい
* 無音状態が続くと認識が終了する

そのため、自然な会話練習というよりも、
「簡潔に、はっきり話す練習」に適したアプリとなっています。

## 学んだこと

本アプリの開発を通して、以下を学びました。

* アクティビティのライフサイクルとリソース管理の重要性
* 音声認識APIの特性と制約
* 状態管理の難しさ

単に機能を実装するだけでなく、
挙動を検証しながら理解することの重要性を実感しました。
